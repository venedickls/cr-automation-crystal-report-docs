COM Automation — CrystalDesignRunTime

Modify existing report objects via the built-in COM server. No extra software needed — works right now.
Easy Free Already installed

What This Approach Can Fix / Automate

✏️
Rename a column headerChange "Remarks" → "Notes", or any existing text label — no Designer needed
↔️
Reposition a columnMove an existing header or field to a different Left/Top coordinate
📐
Resize an objectChange Width and Height of any existing text or field object
🎨
Change font / bold / colorUpdate font size, bold, or foreground color on existing objects
👁️
Show or hide a columnSuppress or un-suppress any section or object
🗄️
Modify the SQL queryChange CommandText SQL — add WHERE, JOIN, new columns on SQL Command-type reports
🔌
Change ODBC connectionSwap the DSN or update stored credentials
📋
Audit report structureList all sections, objects, positions, and text labels
⚠️ Hard limitation: COM can only modify objects that already exist. It cannot create new objects. Use Approach 2 for that.

Can Claude Be Used?

✅ Yes — Claude can fully execute this approach

You describe what you need in plain English. Claude inspects the report structure, identifies the exact object, writes the VBScript, and runs it using the COM server already installed on your machine. No manual scripting required.

Sample prompt — rename a column header
In the report at C:\Crystal Reports Dev\Reports\Active\SBRR_MARKET_UPDATE.rpt, rename the column header labeled "Remarks" to "Notes" in PageHeaderSection1. Use COM automation (CrystalDesignRunTime). Save to the local working copy only — do not touch the network file.
Sample prompt — change the SQL query
Update the SQL in SBRR_MARKET_UPDATE.rpt to add the MARKET column to the SELECT list. Show me the current SQL first before changing it. Use COM automation.
Sample prompt — reposition a column
In SBRR_MARKET_UPDATE.rpt, move the "Cruise Code" column header 200 twips to the right using COM automation. Keep the same font and size.
What Claude does: Reads the report via COM → finds the section and object → inspects the current value → makes the change → saves the local file → confirms before/after. Claude will not touch the network file unless you explicitly say so.

What It Is

Crystal Reports 2016 ships a 32-bit COM server (CrystalDesignRunTime.Application) driveable from VBScript or PowerShell. No extra installs — already on this machine.

Run Command

%SystemRoot%\SysWOW64\cscript.exe //NoLogo "C:\Scripts\modify_report.vbs"

Example — Rename a label & change position

Set app    = CreateObject("CrystalDesignRunTime.Application")
Set report = app.OpenReport("C:\Reports\MyReport.rpt", 1)
For s = 1 To report.Sections.Count
    Set section = report.Sections.Item(CInt(s))
    If InStr(section.Name, "PageHeader") > 0 Then
        For o = 1 To section.ReportObjects.Count
            Set obj = section.ReportObjects.Item(CInt(o))
            Dim txt : txt = ""
            On Error Resume Next : txt = obj.Text : On Error GoTo 0
            If LCase(txt) = "remarks" Then
                obj.Text = "Notes" : obj.Left = 58265 : obj.Top = 0
            End If
        Next
    End If
Next
report.Save : WScript.Echo "Done."

✅ Pros

  • Zero extra software — works today
  • Reliable for renames, repositions, SQL edits
  • Simple VBScript — easy to maintain
  • Batch-process many .rpt files

❌ Cons

  • Cannot create new objects — hard API limit
  • 32-bit only — must use SysWOW64\cscript
  • Hangs if report is open in Designer during run
  • SQL editing only works on SQL Command-type sources

Can Claude Be Used?

✅ Yes — Claude can fully execute this approach

Once the .NET runtime is installed, tell Claude what you need added, renamed, or modified. Claude writes and runs the PowerShell script against the CrystalDecisions API — including creating brand-new objects that Approach 1 cannot do. Claude handles all the technical detail; you just describe the business need.

Sample prompt — add a new column header
Using the CrystalDecisions .NET API, add a new text label called "Market" to PageHeaderSection1 of C:\Crystal Reports Dev\Reports\Active\SBRR_MARKET_UPDATE.rpt. Position it immediately after the "Cruise Code" header. Match the same font (Arial 8pt bold) and height as the other column headers. Save to the local file only.
Sample prompt — add a new data field column
In SBRR_MARKET_UPDATE.rpt, add a database field object for the MARKET column from Snowflake into DetailSection1. Place it directly under the "Market" header added to PageHeaderSection1. Match the width and height of the adjacent fields. Use the CrystalDecisions .NET API.
Sample prompt — bulk rename across multiple reports
Using the CrystalDecisions .NET API, loop through all .rpt files in C:\Crystal Reports Dev\Reports\Active\ and rename any column header labeled "Remarks" to "Notes". List which files were changed and which had no "Remarks" label.
Sample prompt — export to PDF
Open SBRR_MARKET_UPDATE.rpt using the CrystalDecisions .NET API, set the ODBC login (DSN: AMA_SNOWFLAKE_ODBC_CONNECTOR), run the report, and export to PDF at C:\Reports\Output\SBRR_today.pdf.
What Claude does: Loads the CrystalDecisions DLL → opens the report → inspects the section structure → calculates coordinates from existing object positions → generates and runs PowerShell → confirms what was added with exact positions.

Install Steps

  • Go to tools.sap.com → sign in (SAP ID: P2011371697)
  • Download: "SAP Crystal Reports for Visual Studio SP40 CR Runtime 64-bit MSI"
  • Submit IT request to install on Citrix VD
  • Verify: Get-ChildItem "C:\Program Files (x86)\SAP BusinessObjects" -Recurse -Filter "CrystalDecisions.CrystalReports.Engine.dll"

Example — Add new "Market" column header

Add-Type -Path "...\CrystalDecisions.CrystalReports.Engine.dll"
$rpt = New-Object CrystalDecisions.CrystalReports.Engine.ReportDocument
$rpt.Load("C:\Reports\SBRR_MARKET_UPDATE.rpt")

$section = $rpt.ReportDefinition.Sections["PageHeaderSection1"]
$txt = New-Object CrystalDecisions.CrystalReports.Engine.TextObject
$txt.Text = "Market" ; $txt.Left = 58265 ; $txt.Top = 0
$txt.Width = 1995    ; $txt.Height = 240
$txt.Font.Name = "Arial" ; $txt.Font.Size = 8 ; $txt.Font.Bold = $true
$section.ReportObjects.Add($txt)
$rpt.SaveAs("C:\Reports\SBRR_MARKET_UPDATE.rpt") ; $rpt.Close()

✅ Pros

  • Creates new objects — no COM limitation
  • Works with 64-bit PowerShell
  • Free runtime license
  • Export to PDF/Excel/CSV built in

❌ Cons

  • Requires IT install on Citrix VD
  • DLL paths must be correctly referenced
  • SP40 runtime vs CR 2016 SP9 — minor version gap

Power Automate Desktop (RPA)

Record mouse/keyboard actions in the Crystal Reports Designer GUI and replay them. Free, already on Windows 11.
Beginner Free (Windows 11) Fragile

What This Approach Can Fix / Automate

🖱️
Automate any GUI actionAnything you can do manually in Crystal Reports Designer can be recorded and replayed
📥
Insert new text objectsRecord Insert → Text Object → type → position — replay the same position every time
⏰
Schedule report runsOpen a report, refresh data, export to PDF, send via email — on a timer
📤
Export reports automaticallyAutomate File → Export → PDF every night
⚠️ Breaks if window size, screen resolution, or zoom level changes. Not reliable on Citrix remote sessions.

Can Claude Be Used?

⚠️ Partial — Claude assists but cannot execute GUI actions

Claude cannot control the mouse, record clicks, or interact with the Crystal Reports Designer GUI. Power Automate Desktop requires a human to record the initial flow by performing the actions on screen. However, Claude can design the entire flow — writing out the exact steps to record, variables to create, and logic to add — so you follow Claude's instructions while recording.

Sample prompt — design a flow (Claude plans, you record)
Help me design a Power Automate Desktop flow to insert a new text label "Market" into Crystal Reports PageHeaderSection1. Give me the exact step-by-step mouse clicks and keyboard actions I need to record, in the correct order, including where to click and what to type.
Sample prompt — troubleshoot a broken flow
My Power Automate Desktop flow for Crystal Reports stopped working after I changed screen resolution from 1920x1080 to 1440x900. The flow clicks on the wrong place now. What are the best practices to make the flow resolution-independent?
What Claude does: Designs the flow steps, writes documentation, explains the logic, troubleshoots failures. What Claude cannot do: Record the flow, click the mouse, or interact with the Crystal Reports GUI. A person must perform the recording step.

How to Set Up

  • Search "Power Automate" in Windows Start — pre-installed on Windows 11
  • Click New flow → name it → click Record
  • Perform the action in Crystal Reports — every click and keypress is captured
  • Stop recording → review the generated steps
  • Add input variables (report path, column name) to parameterize the flow
  • Run or schedule the flow on demand

✅ Pros

  • Already available — no IT install needed
  • No programming required
  • Can trigger from Teams or Power Automate cloud

❌ Cons

  • Breaks when screen resolution or window changes
  • Hard to parameterize for different column positions
  • Screen must be visible and unlocked during run
  • Not reliable on Citrix remote sessions

SAP BusinessObjects REST API

Manage reports through the BOE platform — schedule, refresh, export. Cannot edit report design or layout.
Intermediate Requires BOE license Limited scope

What This Approach Can Fix / Automate

⏰
Schedule report runsTrigger reports at specific times via API call — no manual scheduling in BOE
📤
Export report outputProgrammatically export to PDF, Excel, CSV and save to a network share
🔄
Refresh data on demandTrigger a data refresh after a nightly Snowflake load
📧
Auto-distribute reportsEmail report outputs to a distribution list after each run
⚠️ The BOE REST API manages report execution only. It cannot add columns, rename labels, or change SQL.

Can Claude Be Used?

⚠️ Partial — Claude writes API scripts but the API cannot change report layout

Claude can write the complete PowerShell or HTTP scripts to authenticate, schedule, export, and distribute via the BOE REST API. However, the BOE API itself has no method to modify report layout — adding columns, renaming labels, or changing SQL is not possible through this API regardless of what Claude writes.

Sample prompt — schedule and export via BOE API
Write a PowerShell script to use the SAP BusinessObjects REST API to authenticate to BOE at http://boe-server:6405, find the report named "SBRR Booking Revenue Report", schedule it to run every weekday at 6:00 AM, and export the PDF output to \\internal\reports\Output\SBRR_daily.pdf.
What Claude does: Writes complete REST API scripts for scheduling and export. What Claude cannot do via this approach: Add columns, rename labels, or change SQL — the BOE API has no endpoints for report design changes. Use Approaches 1 or 2 for those.

Example — Schedule and export

$token = (Invoke-RestMethod -Uri "http://boe-server:6405/biprws/logon/long" `
  -Method Post -ContentType "application/json" `
  -Body '{"userName":"admin","password":"pass","auth":"secEnterprise"}').logonToken

Invoke-RestMethod `
  -Uri "http://boe-server:6405/biprws/infostore/{reportId}/schedules" `
  -Method Post `
  -Headers @{ "X-SAP-LogonToken" = $token } `
  -Body (@{ schedule = @{ recurrence = "once" } } | ConvertTo-Json)

✅ Pros

  • True REST API — integrates with any system
  • Automates scheduling and distribution
  • No local Crystal Reports install needed on server

❌ Cons

  • Cannot edit report design or add columns
  • Requires SAP BOE platform (separate licensed product)
  • Unknown if BOE is deployed in AmaWaterways environment

Direct .rpt Binary Parsing (Python)

Parse the .rpt OLE2 compound document directly — no Crystal Reports software required on the machine.
Expert Free Not production-ready

What This Approach Can Fix / Automate

🔍
Read report metadataExtract structure, object names, and positions without Crystal Reports installed
🔬
Research / reverse engineeringUnderstand the binary .rpt format to eventually build a write-capable parser
⚠️ No mature Python library exists for reliably writing .rpt files. Risk of corrupting reports if write is attempted.

Can Claude Be Used?

❌ Claude cannot add or modify columns via this approach

Claude can write Python to read .rpt binary data using the olefile library (useful for auditing without Crystal Reports installed). However, no Python library exists to reliably write to the .rpt binary format — attempting to do so risks permanently corrupting the report file. Claude cannot use this approach to add columns, rename labels, or change SQL. Use Approaches 1 or 2 for modifications.

Sample prompt — read-only audit (only thing Claude CAN do here)
Write a Python script using the olefile library to open C:\Crystal Reports Dev\Reports\Active\SBRR_MARKET_UPDATE.rpt and list all internal OLE2 streams, then extract any readable text strings (object labels, SQL fragments). Also show me how to install Python and the olefile library on Windows.
What Claude does: Writes the Python read-only parsing script. What Claude cannot do via this approach: Add columns, rename labels, or change SQL — no write library exists for .rpt binary format. For modifications, use Approach 1 or 2.

Example — Read raw .rpt structure

import olefile  # pip install olefile

with olefile.OleFileIO("MyReport.rpt") as ole:
    for entry in ole.listdir():
        print(entry)
    if ole.exists("CRYSETIN"):
        data = ole.openstream("CRYSETIN").read()
        print(f"Stream size: {len(data)} bytes")

✅ Pros

  • No Crystal Reports software needed
  • Cross-platform — could run in cloud/Linux

❌ Cons

  • No working write library exists
  • Binary format is proprietary and undocumented
  • High risk of corrupting .rpt files if write attempted
  • Python not installed on this machine

SSRS — SQL Server Reporting Services

Replace Crystal Reports with Microsoft's reporting platform. Reports are XML (RDL) — fully scriptable by AI or code.
Large migration Included w/ SQL Server Long-term project

What This Approach Can Fix / Automate (after migration)

➕
Add columns by editing XMLAdding a column is inserting XML nodes — Claude can do this instantly
✏️
Rename any labelFind-and-replace in RDL XML — batch-update all reports at once
🗄️
Change SQL directlySQL queries are plain text inside the XML — easy to find and edit
🌐
Web-based deliveryUsers view in browser — no Crystal Reports license for viewers
📦
Version control in GitRDL files are XML — stored in Git and diff'd like source code

Can Claude Be Used?

✅ Yes — Claude can fully modify SSRS RDL files (easiest AI integration)

Because SSRS reports are plain XML (.rdl files), Claude reads, modifies, and writes them directly — no special software or COM/DLL needed. Adding a column, renaming a label, changing SQL — all pure XML text. This is the approach where Claude has the fewest limitations. After migration, every report change request becomes a simple prompt.

Sample prompt — add a column to an RDL file
Open the SSRS report at C:\Reports\SSRS\SBRR.rdl and add a "Market" column after "CruiseCode". The header should be bold. The data cell should bind to the MARKET field from the dataset. Match the width of the CruiseCode column. Save the file.
Sample prompt — batch rename across all RDL files
Loop through all .rdl files in C:\Reports\SSRS\ and rename any column header text "Remarks" to "Notes". List which files were changed.
What Claude does: Reads the RDL XML → finds the relevant nodes → inserts or modifies the XML → saves the file. No DLL, no COM, no software required — pure text editing.

Adding a column via PowerShell (XML)

[xml]$rdl = Get-Content "C:\Reports\SBRR.rdl"
$ns = "http://schemas.microsoft.com/sqlserver/reporting/2010/01/reportdefinition"
$newCell = $rdl.CreateElement("TablixCell", $ns)
$newCell.InnerXml = @"
  <CellContents xmlns='$ns'>
    <Textbox Name='txtMarket'>
      <Value>Market</Value>
      <Style><FontWeight>Bold</FontWeight></Style>
    </Textbox>
  </CellContents>
"@
$rdl.Save("C:\Reports\SBRR.rdl")

✅ Pros

  • Reports are XML — trivially scriptable by Claude
  • Web viewer — no desktop software for end users
  • Included with SQL Server
  • Version control via Git

❌ Cons

  • All existing Crystal Reports must be rebuilt in SSRS
  • Fewer pixel-perfect formatting options
  • Requires SQL Server instance or Azure SQL
  • 6–18 month migration project

Easy for new reports $10/user/mo Pro Long-term project Best long-term option

What This Approach Can Fix / Automate (after migration)

🤖
AI Copilot for column additionsType "Add a Market column" — Copilot builds the visual and data binding automatically
🔌
Native Snowflake connectionNo ODBC DSN needed — built-in Snowflake connector
🗄️
Modify SQL in Power QueryEditable in Power Query editor — add columns, filters, joins
🌐
Browser-basedNo Crystal Reports license for viewers
🔄
Automatic data refreshSchedule daily/hourly refresh from Snowflake
🤝
Teams integrationEmbed live reports directly in Microsoft Teams channels
⚙️
REST API for automationPush datasets, trigger refreshes, manage permissions programmatically

Can Claude Be Used?

✅ Yes — Claude writes all Power Query M, DAX, and REST API code

Claude can write Power Query M code (add/transform columns from Snowflake), DAX measures (calculated metrics), and Power BI REST API scripts (trigger refreshes, push data, manage permissions). Claude cannot open the Power BI Desktop app or the browser-based Power BI Service directly, but it writes all the code — you paste it in or run the script.

Sample prompt — add a column in Power Query M
Write the Power Query M code to connect to our Snowflake instance (amawaterways.snowflakecomputing.com, warehouse AMA_WAREHOUSE, database AMA_DB, schema REPORTING) and load the BOOKINGS_REVENUE table. Add a calculated column "Net Revenue per Guest" = [GROSS_REVENUE] / [GUEST_COUNT]. Show me where to paste this in Power BI Desktop.
Sample prompt — trigger dataset refresh via REST API
Write a PowerShell script using the Power BI REST API to trigger a manual dataset refresh for the "SBRR Bookings Revenue" dataset in the "IT Reports" workspace. Use a service principal for authentication (tenant ID, client ID, client secret from environment variables). This should run automatically after each nightly Snowflake data load.
What Claude does: Writes complete Power Query M, DAX, and REST API code — copy-paste ready. What Claude cannot do: Directly open Power BI Desktop or the Power BI Service UI. You paste Claude's code into Power BI, or run the PowerShell REST API script.

Snowflake connection in Power Query

let
    Source = Snowflake.Databases(
        "amawaterways.snowflakecomputing.com",
        "AMA_WAREHOUSE"
    ),
    DB     = Source{[Name="AMA_DB"]}[Data],
    Schema = DB{[Name="REPORTING"]}[Data],
    Table  = Schema{[Name="BOOKINGS_REVENUE"]}[Data]
in
    Table

✅ Pros

  • Native Snowflake connector — no ODBC config
  • AI Copilot built in
  • Web-based — end users need no software
  • REST API for full programmatic automation
  • Teams integration, automatic refresh

❌ Cons

  • $10/user/month for Pro license
  • All existing Crystal Reports must be rebuilt
  • Less suited for pixel-perfect print reports
  • 6–12 month migration project

JasperReports — Open Source Java Engine

Replace Crystal Reports with a fully open-source reporting engine. Reports are JRXML (XML) — fully programmable.
Advanced Free (Community) Long-term project

What This Approach Can Fix / Automate (after migration)

➕
Add columns by editing JRXMLReports are XML — adding a column is inserting a few XML elements, scriptable by Claude
✏️
Rename any labelFind-and-replace in XML — instant across all reports
🗄️
Modify SQL data sourceSQL queries are plain text inside JRXML
🖨️
Pixel-perfect print formattingProduces print-quality output like Crystal Reports
⚙️
REST API via JasperReports ServerRun, schedule, and export reports via HTTP
💰
Zero license costCommunity edition is completely free

Can Claude Be Used?

✅ Yes — Claude can fully modify JRXML files (same as SSRS — pure XML)

Just like SSRS, JasperReports uses XML (.jrxml). Claude reads, modifies, and writes JRXML files directly — no Java, no Jasper Studio required. Adding columns, renaming labels, changing SQL — all pure XML that Claude handles natively. After migration, any report change is a simple prompt.

Sample prompt — add a column to a JRXML file
Open C:\Reports\Jasper\SBRR.jrxml and add a "Market" column after "CruiseCode". Header text should be bold Arial 8pt. The data cell should bind to $F{MARKET}. Match the width of the CruiseCode column. Save the file.
Sample prompt — generate a new JRXML report from scratch
Create a new JasperReports JRXML file for a "SW2 Sailings Summary" report with columns: Sailing Code, Guest Count, Gross Revenue, Market. The Snowflake data source is amawaterways.snowflakecomputing.com. Include a page header with the title and page footer with the page number. Save to C:\Reports\Jasper\SW2_Summary.jrxml.
What Claude does: Reads the JRXML → modifies the XML → adds staticText (header) and textField (data) elements → updates the SQL queryString → saves the file. No Jasper Studio, no Java compilation needed.

Adding a column in JRXML

<textField>
  <reportElement x="580" y="0" width="100" height="20"/>
  <textElement><font fontName="Arial" size="8" isBold="true"/></textElement>
  <textFieldExpression><![CDATA[$F{MARKET}]]></textFieldExpression>
</textField>

✅ Pros

  • Completely free and open source
  • JRXML is XML — Claude can modify natively
  • Pixel-perfect output like Crystal Reports
  • Runs on any OS, no per-user license

❌ Cons

  • Java knowledge required for server setup
  • All Crystal Reports must be rebuilt
  • Smaller Microsoft ecosystem integration
  • 12–24 month migration project

  • Existing Crystal Reports → maintained via Approaches 1–2 until natural retirement
